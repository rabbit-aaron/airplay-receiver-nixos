# AirPlay 2 Receiver on NixOS (Raspberry Pi 3)

NixOS configuration for a Raspberry Pi 3 running as an AirPlay 2 receiver using [shairport-sync](https://github.com/mikebrady/shairport-sync) and a HiFiBerry DAC (or PCM5102A-based clone).

## Features

- AirPlay 2 with multi-room speaker grouping
- HiFiBerry DAC / PCM5102A support via device tree overlay (`simple-audio-card`)
- Line-out mode (fixed max volume, speaker handles volume control)
- Low-latency buffer settings for snappy playback
- Firewall enabled with only required ports open

## Hardware

- Raspberry Pi 3 Model B
- HiFiBerry DAC (or PCM5102A-based clone)

## Files

| File | Description |
|------|-------------|
| `configuration.nix` | Main NixOS system configuration |
| `hifiberry-dac.dts` | Device tree overlay for PCM5102A DAC via I2S |
| `config.txt` | Raspberry Pi firmware config (overclock, disable onboard audio) |
| `wireless.nix.example` | Template for WiFi credentials |

## Setup

1. Flash NixOS aarch64 SD image for Raspberry Pi 3
2. Copy files to `/etc/nixos/`
3. Create `wireless.nix` from the example:
   ```
   cp wireless.nix.example wireless.nix
   # Generate your PSK:
   wpa_passphrase "YourSSID" "YourPassword"
   # Paste the psk= value into wireless.nix as pskRaw
   ```
4. Copy your `hardware-configuration.nix` (generated by `nixos-generate-config`)
5. `sudo nixos-rebuild switch`

## Ports

The firewall opens the following ports for AirPlay 2:

| Port | Protocol | Purpose |
|------|----------|---------|
| 22 | TCP | SSH |
| 3689 | TCP | DACP remote control |
| 5000 | TCP | AirPlay HTTP |
| 7000 | TCP | AirPlay 2 RTSP |
| 319, 320 | UDP | PTP timing (NQPTP) |
| 5353 | UDP | mDNS / Bonjour |
| 6000-6009 | UDP | RTP audio streaming |
| 32768-60999 | TCP/UDP | Ephemeral ports for AirPlay 2 |

## Device Tree Overlay

The `hifiberry-dac.dts` overlay uses the `simple-audio-card` kernel framework rather than the HiFiBerry machine driver (which is not in the mainline kernel). It configures:

- I2S GPIO pin muxing (GPIO 18-21)
- PCM5102A codec binding
- `simple-audio-card` sound device

This approach works with any PCM5102A-based DAC board, not just the official HiFiBerry DAC.
